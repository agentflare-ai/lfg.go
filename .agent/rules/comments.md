**Comments**
- NEVER leave TODO, FIXME, or thought comments in generated code
- NEVER comment obvious code—prefer self-documenting names and structure
- NEVER leave commented-out code blocks
- ALWAYS use doc comments (`///` or `/** */`) for public APIs explaining purpose, parameters, and return values
- ALWAYS document non-obvious invariants, constraints, or assumptions
- ALWAYS document why, not what—code shows what, comments explain reasoning
- NEVER duplicate in comments what the code already clearly expresses
